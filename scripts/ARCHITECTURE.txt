================================================================================
  数据流: 单个令牌经过完整流水线
================================================================================

  请求到达 (泊松分布, 均值 λ req/s)
      │
      ▼
  [调度器] 接纳等待请求 → 状态=预填充(PREFILL)
      │
      │  组建预填充批次, 提交给模型执行器
      ▼
  [模型执行器] execute_batch(prefill)
      │
      ├─► [KV缓存管理器] allocate_prefill(提示词长度, 芯片列表)
      │       └─► 在每个芯片上分配KV块 (~2块/芯片, 512 token提示词)
      │
      └─► [流水线引擎] 处理全部61层:
              对每层 L=0..60:
                芯片 = 集群.get_chip_for_layer(L)
                │
                ├─ 1. 权重预取:     SRAM确定性权重 (0.1us)
                ├─ 2. MLA注意力:    DSP计算 (~9.7us/芯片, 批次缩放)
                ├─ 3. 注意力归一化: DSP微量 (~0.001us)
                ├─ 4. MoE路由器:    选择top-6专家 (DSP+SRAM)
                ├─ 5. MoE分发:      C2C发送至专家芯片 (并行最大延迟)
                ├─ 6. 共享专家:     DSP计算 (始终本地)
                ├─ 7. 路由专家:     DSP+HBM (仅本地专家, 0/1/2命中)
                ├─ 8. MoE汇聚:      C2C接收专家输出
                ├─ 9. FFN归一化:    DSP微量
                └─10. 流水线转发:   C2C转发至下一层的芯片
                      │
                      ▼
                 (同卡跨芯片) C2CDualRing.transfer_time_us()
                      │
                      ▼
                 (跨卡)     PCIeFabric.transfer_time_us()
      │
      ▼
  [批次完成事件] → on_prefill_complete() → 状态=解码(DECODE)
      │
      ▼
  [模型执行器] execute_batch(decode, batch_size=N)
      │
      ├─► [KV缓存管理器] allocate_decode(每16步分配1块/芯片)
      ├─► [流水线引擎] throughput_model(N): TPS = 17445×N/(N+25.4)
      └─► 耗时 = latency_model(N) = 1e6/TPS
      │
      ▼
  [批次完成事件] → on_decode_step()
      │
      ├─ 已生成令牌数 += 1
      ├─ 若已完成: 状态=完成(FINISHED), free_kv_blocks()
      └─ 否则: 停留解码态, ~1ms后进入下一批次

================================================================================
  MoE 专家路由流程
================================================================================

  令牌在层L, 由芯片A (例如 C0.0) 处理
      │
      ▼
  路由器选择top-6专家: [E50, E120, E200, E300, E350, E5]
      │
      ├─ 本地 (芯片A):   E5    (12专家/芯片/384 = 3.1%)
      ├─ 同卡远程:       E50   (芯片 C0.1) → C2C 环A (1跳)
      ├─ 跨卡远程1:      E120  (芯片 C1.0) → C2C + PCIe P2P
      ├─ 跨卡远程4:      E200  (芯片 C4.0) → C2C + PCIe P2P
      ├─ 跨卡远程7a:     E300  (芯片 C7.1) → C2C + PCIe P2P
      └─ 跨卡远程7b:     E350  (芯片 C7.2) → C2C + PCIe P2P

  分发 (并行): 250ns × max(1本地 + 5远程跨芯片, 跨卡延迟)
  汇聚 (并行): 250ns × max(1本地 + 5远程跨芯片, 跨卡延迟)

  专家命中分布:
    P(0个本地) = (1 - 12/384)^6 = 82.7%  → 全部远程 (最坏情况)
    P(1个本地) = 6 × 12/384 × (1-12/384)^5 = 16.5%  → 1本地 + 5远程
    P(2+本地)  = 0.8%  → 2个以上本地 (罕见, 可节省额外C2C跳)


================================================================================
  关键架构决策
================================================================================

  1. 分布式层流水线 (非单芯片复制)
     - 61层分布到32芯片
     - 每层在分配的芯片上处理令牌
     - C2C流水线转发传输隐藏状态到下一层
     - 流水线深度 = 32 (同时有32个令牌在不同阶段飞行)

  2. 双路径时序模型
     - 快速路径: throughput_model(B) = 17445×B/(B+25.4), O(1)供调度器使用
     - 详细路径: 逐阶段 DSP/HBM/C2C 时序, 供深入分析

  3. 批次感知计算
     - DSP MAC 随 batch_size 线性缩放 (Q/KV/O投影和点积)
     - HBM 权重加载批次内共享 (只加载一次)
     - C2C 分发使用组播 (每个目标芯片一次传输)

  4. PagedAttention KV 缓存
     - 块粒度: 16 tokens/块, 576 bytes/token (FP8 K+V)
     - 逐芯片分配, 按层比例 (~2/61 每芯片)
     - HBM满时 LRU 淘汰

  5. MoE 散射-汇聚
     - 384专家分布: 每芯片12个 (连续编号范围)
     - 专家权重存HBM (每个33 MB, fp4)
     - 远程专家通过C2C分发/汇聚 (并行)

  6. 权重放置策略
     - 确定性权重 (注意力+共享专家+路由器+归一化): SRAM, 双缓冲
     - 专家权重: HBM, 计算时按需流式加载
     - 总权重: ~0.7 GB/芯片 (fp4, 32芯片分摊)
