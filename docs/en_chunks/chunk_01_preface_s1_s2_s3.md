## 1. 战略定位与必要性

### 1.1 核心问题

中国大模型（以 DeepSeek 系列为代表）面临一个结构性矛盾：

```
  模型能力全球领先 (DeepSeek V4 Pro 1.6T MoE)
         ↕
  推理部署硬件受制于供应管制
         ↕
  ┌─────────────────────────────────────────┐
  │ 可用硬件          受制因素                │
  ├─────────────────────────────────────────┤
  │ NVIDIA H100/B200  美国出口管制 (3A090)   │
  │ AMD MI300X        同等管制               │
  │ 华为 Ascend 910C  SMIC 7nm产能 + CoWoS   │
  │                   先进封装制裁 + 国际封堵 │
  │ 寒武纪/海光/壁仞   不同组合的供应/生态问题 │
  └─────────────────────────────────────────┘
```

### 1.2 FPGA 路径的独特优势

```

  FPGA 芯片:      Intel/Altera Fab (美国/爱尔兰/以色列)

  EMIB 封装:       Intel 马来西亚 / 越南
  PCB 制造:        中国大陆 / 台湾
  PCIe 标准:      全球通用, 不受管制
  Quartus 工具:   全球可用 (非武器级软件)

  Altera 独立运营 (2024):
    从 Intel 剥离为独立公司 → 财务独立, FPGA 为唯一业务
    → 不受 Intel 代工亏损拖累, PE 入股筹备 IPO
    → Agilex 产品线生命周期 10-15 年 (业界惯例)
    → 32 片订单属常规客户, 交期 8-12 周

  关键特征:
  ┌────────────────────────────────────────────┐
  │ ✓ 不依赖 SMIC 产能                         │

### 1.3 战略定位：三阶段路线 (FPGA → eASIC → ASIC)

> **Phase 1 — FPGA 原型验证 (现在):** 低风险、低 NRE 验证 fp4+MLA 架构，服务种子客户。
> **Phase 2 — FPGA 量产 + eASIC 转换:** FPGA 量产交付出货 + eASIC 结构化降本 (同工艺，仅换金属层)。
> **Phase 3 — 全定制 ASIC (远期):** 市场验证 >1,000 套/年后，HBM3@7nm 终极降本。

```

### 1.4 技术路线论证：为什么是 FPGA→ASIC 两阶段？

**评审质疑: "科研方向是否正确？为什么选 FPGA 而不是直接 ASIC / GPU 优化 / 等待 Chiplet？"**

答: 不是 "FPGA 或 ASIC", 是 "先用 FPGA 验证, 再流 ASIC 量产"。两者是同一个路线的两个阶段。

**一、四条可选路线的矩阵对比**

```
┌────────────────────┬──────────────┬────────────┬──────────┬──────────┐
│                     │ FPGA→ASIC    │ 直接自研ASIC│ GPU优化   │ 等待Chiplet│
│                     │ (本方案)     │            │ (CUDA)   │          │
├────────────────────┼──────────────┼────────────┼──────────┼──────────┤
│ 第一阶段投入         │ ~¥7M (FPGA) │ ~¥50-200M  │ ~¥5M     │ ~¥3M     │
│ 到可部署时间         │ ~10 个月     │ 18-24 个月 │ 6-12 个月│ 不确定    │
│ 架构验证风险         │ 低 (FPGA重配)│ 高 (流片固化)│ 低      │ 高        │
│ 终局硬件成本         │ ~$150-190K/套│ ~$50K/套  │ ~$280K    │ 未知      │
│ fp4 原生             │ ✓           │ ✓ 可硬化   │ ✗        │ △        │
│ MLA 硬化             │ ✓           │ ✓ 可硬化   │ ✗        │ ✗        │
│ 全球可部署           │ ✓           │ ✓ (自研)   │ ✗ 管制   │ ✓        │
│ 模型演进适应性        │ ✓ FPGA可重配│ ✗ 固化     │ ✓ 灵活   │ ✓ 灵活    │
│ 供应链多源            │ ✓           │ △ 单一     │ ✗ TSMC独 │ ✗ TSMC独  │
└────────────────────┴──────────────┴────────────┴──────────┴──────────┘

关键差异:
  直接 ASIC:  一次流片押注 → 失败 = $50-200M 打水漂
  FPGA→ASIC: FPGA 先跑通 → 真实数据验证 → 流片风险极低
             RTL 复用率 >70% → ASIC 设计周期从 18 月缩至 12 月
```

**二、为什么不是直接 ASIC？**

```
直接 ASIC 在这个阶段不可行:

1. 架构风险未释放:
   ● fp4+MLA 推理架构在硅层面未经任何验证
   ● DeepSeek V5/V6 可能改变 MoE Top-K、MLA 压缩算法
   ● 直接流片 → 架构变更 = $10-20M 改版费
   ● FPGA 先行: 架构变更 = 重新编译 (8h, $0)

2. 没有 workload 数据:
   ● ASIC 设计需要真实负载数据做 microarchitecture 优化
   ● 没有种子客户跑过 → ASIC 设计靠猜 → 性能和面积都不在最优点
   ● FPGA 先跑 10 套种子客户 → 积累数据 → 指导 ASIC 设计

3. 资金节奏:
   ● FPGA 阶段 ¥6.75M → 可承受
   ● Phase 3 种子客户收入 → 可部分覆盖 ASIC NRE
   ● 不是一次性赌 $50M+, 而是分阶段释放风险
```
   → 10 个月 FPGA 验证窗口刚好卡在 "现在需要 + 未来不确定" 之间

3. 不互斥:
   ● FPGA RTL 代码 ~60% 可直接用于 ASIC 设计
   ● 如果市场验证成功 (100+ 套)，FPGA→ASIC 是自然演进
   ● 如果市场验证失败，¥7M 研发沉没 vs ¥200M ASIC NRE
   → FPGA 是 ASIC 的 "市场验证前置"，不是 "二选一"

科研界类比: FPGA 推理 != "放弃 ASIC"， 
            FPGA 推理 = "用可重构硬件验证架构假设，降低 ASIC 投片前的风险"
            类似 Google 用 FPGA 验证 TPU 架构，然后才流片。
```

**三、为什么不是 "优化 GPU CUDA kernel 就够了"？**

```
这是最常见的质疑。但忽略了三个关键事实:

1. GPU 买不到 — 这不是成本问题，是 0 供给问题:
   ● 本文不是和 GPU 比 "谁更好"，GPU 当然更成熟
   ● 本文的场景是 "GPU 供给了 0，你有别的方案吗？"
   ● 优化 CUDA kernel 的前提是有一张 GPU 可以跑 — 目标客户没有这个前提

2. fp4 在 GPU 上是模拟的:
   ● NVIDIA 无 fp4 Tensor Core (H100/B200 最细粒度 INT8/FP8)
   ● GPU 跑 fp4 需要加载 fp4 → 解压到 FP8 → Tensor Core 计算
     解压开销 ~10-15% 带宽 + 延迟
   ● FPGA 直接在 DSP 链上以 fp4 计算，零解压
   → 即使有 GPU，fp4 推理也不是 GPU 的最优路径

3. MLA 在 GPU 上是软件实现:
   ● GPU 跑 MLA 需要 4 个 CUDA kernel: Q projection → KV decompress →
     attention → O projection，中间多次显存往返
   ● FPGA 将 MLA 硬化为一个流水线，KV 不离开片上 SRAM
   → 这不是 "优化 kernel"，这是 "消除 kernel 间的数据移动"
   → GPU 的天花板被 HBM 带宽限制，FPGA 的天花板是 DSP 算力

类比: 在 GPU 被管制的前提下讨论 CUDA 优化，
      就像在禁运名单上讨论 "去哪买打折 H100" — 问题不是优化，是准入。
```

**四、为什么不是等待 Chiplet / 存内计算 / 光学计算？**

```
这些是值得关注的技术方向，但它们的时间线和成熟度不适合当前决策:

  Chiplet AI:
    ● UCIe 标准仍在演进，量产级 Chiplet AI 芯片 (如 d-Matrix) 2026 年才送样
    ● 需要 TSMC CoWoS 或类似封装 — 同样受先进封装制裁
    → 3-5 年后可能成熟，现在不能等

  存内计算 (PIM / Compute-in-Memory):
    ● Samsung HBM-PIM 已发布但无公开购买渠道
    ● 编程模型完全不同于现有 AI 框架，适配成本未知
    → 实验室阶段，不适用于 2 年内需要部署的场景

  光学计算 (Lightmatter / Lightelligence):
    ● 目前仅演示了矩阵乘的原型，离完整 LLM 推理还很远
    ● 精度限制 (模拟计算)，精度损失与 fp4 类似但无成熟校准方案
    → 5-10 年才可能实用化

FPGA 是唯一 "今天可以买到、10 个月可以部署、不受管制" 的选项。
不是否定了前沿方向，而是 FPGA 恰好是前沿 (fp4+MLA 硬化) 和现实 (可获取) 的交点。
```

**五、这个科研方向的学术/工业站位**

```
本方案在三个层级上有独立的研究贡献:

1. 系统架构层面:
   ● 首次提出 "MoE Layer-Wise Pipeline over Ethernet" 的 FPGA 集群方案
   ● 不是 "把 GPU 架构搬到 FPGA"，而是利用 FPGA 的可重构性
     重新设计推理数据路径 (Weight Stationary + Systolic Array)

2. 数值计算层面:
   ● 在 FPGA 上实现 fp4 原生推理 (业界尚无公开的完整方案)
   ● 验证 fp4 在 MoE+MLA 架构下的端到端精度 (对标 BF16 baseline)
   → 如果验证通过，将确立 FPGA 在低精度推理的独特优势

3. 硬件-算法协同设计层面:
   ● MLA 的 KV 压缩 → 直接映射到 FPGA 的矩阵乘法流水线
   ● MoE 的 Expert Routing → 映射到 FPGA 的分布式路由逻辑
   ● Router 的 fp8 例外 → 映射到 FPGA 的混合精度计算链
   → 这不是 "优化已知算法"，是 "从硬件特性出发重新设计推理路径"

科研意义: 如果成功，这不是一篇 "FPGA 加速 LLM" 的增量论文，
         而是确立了一个新的推理范式 —
         "可重构、可出口、精确适配单一模型家族的推理硬件路线"。
         这个范式对 GPU 禁运下的中国 AI 产业有系统性价值。
```

### 3.1 MLA 核心特征

```
标准 MHA (如 LLaMA):
  Q/K/V: 各自 128 heads × 128 dim = 16,384 dim per head group
  KV Cache per token: 2 × 128 × 128 × FP16 = 32 KB

DeepSeek MLA:
  KV 压缩为 1 个 latent vector (512 dim) + decoupled RoPE (64 dim)
  KV Cache per token: (512 + 64) × FP8 = 576 Bytes
  
  压缩比: 32 KB → 576 B = 56× 压缩
  这是 DeepSeek 长 context 能力的核心基础
```
