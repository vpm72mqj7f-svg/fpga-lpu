## 七、与云 GPU 租赁对标 (缺失的关键对比)

评审指出: 客户可以不买 GPU，直接租云 GPU 跑推理。本节补上这个缺失的对标。

**8×H100 云实例推理成本:**

```
主流云厂商 8×H100 实例 (2025-2026):
  AWS p5.48xlarge:     ~$15-20/hr (按需)
  Lambda Labs 8×H100:  ~$12-16/hr (按需)
  国内云 (存量 H100):    ~¥50-80/hr (极其稀缺)

H100 跑 DeepSeek V4 Pro Decode:
  吞吐 (估计):              ~600 tok/s (B=1~4, MoE 下 batch 有限)
  每小时产出:               600 × 3600 = 2.16M tok
  $/百万 token:            $15 / 2.16M = $6.9/百万token (按 $15/hr)
                          $20 / 2.16M = $9.3/百万token (按 $20/hr)
```

**与 FPGA 方案的硬件 TCO 对比:**

```
┌─────────────────────────┬────────────┬────────────┬────────────┬────────────┐
│                          │ 云 GPU 租赁 │ FPGA 10套  │ FPGA 100套 │ FPGA 10K套 │
│                          │ 8×H100     │ (原型)     │ (批量)     │ (规模)     │
├─────────────────────────┼────────────┼────────────┼────────────┼────────────┤
```

**云 GPU 价格上涨趋势:**

```
H100 云实例价格变化 (8卡节点, $/hr):
  2024 Q1:  $2-3/hr   (供应充裕)
  2024 Q3:  $5-8/hr   (需求暴增)
  2025 Q1:  $8-12/hr  (B200 延期 + 管制收紧)
  2025 Q4:  $12-20/hr (H100 停产, 存量见底)
  2026:     $15-25/hr (趋势上涨, 稀缺性溢价)

深层原因:
  ● H100 已停产 (2024 Q4), 存量卡逐渐退役
  ● B200 价格更高 ($30-40K/卡) 且同样管制
  ● 云厂商优先供给训练大客户, 推理散户拿不到
  ● 管制收紧: 中东、部分东南亚已被限制

结论: 云 GPU 租赁的 $/百万token 正在快速逼近甚至超越 FPGA 量产方案。
      当云 GPU 涨到 $25/hr → $11.6/百万token, 已高于 FPGA 的 $7-9。
```

**产能/利用率对 FPGA 成本的敏感性 (10套集群):**

```
┌──────────────────┬──────────┬──────────┬──────────┬──────────┐
│ 利用率            │ 30%      │ 50%      │ 70%      │ 90%      │
├──────────────────┼──────────┼──────────┼──────────┼──────────┤
│ 年产出 (B tokens) │ 7.6      │ 12.6     │ 17.7     │ 22.7     │
│ $/百万token       │ $14      │ $8.5     │ $6.0     │ $4.7     │
│ (量产 100 套)    │ $11.7    │ $7.1     │ $5.0     │ $3.9     │
└──────────────────┴──────────┴──────────┴──────────┴──────────┘

与云 GPU 的交叉点 (量产 10 套):
  ● 云 GPU @$9/百万token: FPGA 需 >39% 利用率即可打平
  ● 云 GPU @$12/百万token: FPGA 只需 >25% 利用率
  ● 云 GPU @$20/百万token: FPGA 在任何利用率下都赢
```

**核心结论:**

云 GPU 租赁在 2024 年确实比 FPGA 便宜很多 ($3/hr → $1.4/百万token)，但 2025-2026 年价格快速上升后，FPGA 量产方案的硬件 TCO $3.8-6.0/百万token (baseline) 或 $0.74-1.25/百万token (优化口径) 已经全面优于云 GPU 租赁。对无法合规获取云 GPU 的中国客户来说，这个对比本身就是伪命题——FPGA 是唯一选项。

更关键的是: 云 GPU 租赁不能解决 **数据不外传** 和 **海外部署受管制** 两个硬约束。金融、医疗、政府的私有化部署场景天然排斥公有云 API。FPGA 对标的是 "买不到 GPU + API 不可用" 场景，而不是 "能按小时租 H100" 的场景。

注: 以上 $/token 为纯硬件折旧 + 运营成本，不含 IP License (客户可趸交 ¥2-5M 一次性买断)。含 IP 的 $/百万token 在 10 套级约为 $6.7, 100 套及以上 IP 摊薄可忽略。

## 八、适用场景与价值定位

FPGA 集群的竞争场景不是公共 API 价格战，它的价值在：

```
场景 A: 隐私合规私有化部署
  客户 (金融/医疗/政府) → 数据不能出内网
  公共 API → 不可用
  私有 GPU 集群 → 买不到高端 GPU
  私有 Ascend 集群 → 排队等货
  FPGA 私有集群 → 可获取, 可部署
  
  此时对标不是 DeepSeek API 的 $1.46/百万token,
  而是 "买不到 GPU 导致业务无法上线" 的损失。

场景 B: 海外出海部署
  中国模型 → 需要部署到东南亚/中东/拉美
  公共 API 从中国直连 → 延迟/合规问题
  当地建 Ascend 集群 → 华为出口限制
  当地买 NVIDIA GPU → 管制 + 成本
  FPGA 集群 → 标准 IT 设备, 不受管制

场景 C: 自有流量变现
  有稳定用户群 (如 1000 万 DAU),
  日均推理需求 50B tokens,
  公共 API 年费: 50B × 365 / 1M × $1.46 = $26.6M/年
  自建 FPGA 集群 (10 套): ¥2.0M × 10 = ¥20M ≈ $2.8M (一次性硬件售价)
  + ¥0.44M/年 × 10 = ¥4.4M ≈ $0.6M/年 (运营)
  → 首年回本, 之后边际成本极低
  
  对于真正有规模化推理需求的公司,
  自建 FPGA 集群是一个可计算的 Capex vs Opex 决策。
```

## 九、关键结论

```
1. FPGA 方案 $/百万token 随规模递减 (修正后, $2,500/芯片):
   baseline 口径 (单 session 800 tok/s):
     10 套 (原型):  $6.0/百万token (70% util)
     100 套 (批量): $5.0/百万token
     10K 套 (规模): $3.8/百万token
   优化口径 (§4.6.1+§4.8.x 多 session, 混合 4,556 tok/s):
     10 套:  $1.25/百万token
     100 套: $0.94/百万token
     10K 套: $0.74/百万token

2. 与云 GPU 租赁对标 (baseline 口径):
   云 GPU @$7/M:  FPGA 10 套已优于 ($6.0)
   云 GPU @$10/M:  FPGA 任何规模都优于
   云 GPU @$15/M:  FPGA 任何规模都碾压

3. 与公共 API 对标 (优化口径):
   DeepSeek V4 Pro API @$1.46/M: FPGA 100 套及以上更便宜 ($1.30, $1.03)
   OpenAI API @$2-15/M:  FPGA 10 套即优于 ($1.25)
   → FPGA 修正口径在所有规模下都优于公共 API 价格

4. 芯片定价 ($2,500 基准):
   10 套: $2,500/ea (320 片), 100 套: ~$1,950/ea (3,200片), 10K 套: ~$1,390/ea (320K片)
   外围物料 1:1 → 1:0.7 → 1:0.5 随量递减

5. 核心差异不在 ¥/token 的最低值, 而在:
   → 可获取 (NVIDIA 买不到)
   → 可海外部署 (Ascend 出不去)
   → 数据不外传 (API 不行)
   → 供应链多源 (不被卡脖子)
```

## 附表：归一化计算核心公式

```
$ / 百万 token =  TCO_annual / (Throughput × 86400 × 365 × Utilization / 1e6)

其中:
  TCO_annual   = Hardware / 3 + R&D / 5 + Opex
               = ¥495K + ¥450K + ¥273K = ¥1,218K (10 套, $2,500/芯片基准)

  Throughput   = 800 tokens/s (baseline, 保守) 或 4,556 tokens/s (优化口径混合负载)

  Utilization  = 0.7 (日 16h 高峰 + 8h 低谷)

  $ / 百万token = 2,550,000 / (800 × 86,400 × 365 × 0.7 / 1e6)
               = 2,550,000 / (17,671M)
               = ¥110 / 百万token
               ≈ $20 / 百万token
```
